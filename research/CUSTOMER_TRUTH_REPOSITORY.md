# Customer Truth Repository

Imaging Agriculture

Version 2.0

---

# Philosophy

機能要望を集めるな。

ユーザーの真実を蓄積せよ。

---

真実とは:

行動が変わったこと。
見え方が変わったこと。
感情が動いたこと。

「この機能がほしい」は真実ではない。
「旅行中に畑を見るようになった」が真実。

---

# 保存対象

```
✅ 行動変化
   「散歩中に草地が気になるようになった」
   「旅行中に畑を見るようになった」
   「雑草を抜かずに観察するようになった」

✅ 認識変化
   「雑草は情報だと思うようになった」
   「土地の見え方が変わった」
   「農地を見て何かを想像できるようになった」

✅ 感情変化
   「土地を見るのが楽しくなった」
   「畑の前で立ち止まるようになった」
   「自分に農業が向いているか考えるようになった」

✅ 失敗シグナル（記録して原因を分析する）
   「勉強になりました」
   「クイズみたいで楽しかった」
   「正解できて嬉しかった」
```

---

# 保存しないもの

```
❌ 機能要望
   「シェア機能がほしい」
   「コメントできるといい」
   「通知がほしい」
   → ユーザーが本当に求めているものの表面。深く聞く。

❌ 褒め言葉
   「すごいアプリですね」
   「よくできていますね」
   → 体験価値の確認にならない

❌ 感想だけ
   「面白かった」
   → 「何が面白かったか」「その後どう変わったか」を掘り下げる
```

# タグ定義

```
#行動変化    — 外の世界での行動が変わった
#認識変化    — 見え方・考え方が変わった
#感情変化    — 感情・好奇心が変わった
#継続動機    — アプリを使い続ける理由になっている
#離脱        — 使わなくなった理由
#失敗シグナル — 学習アプリ・クイズアプリ化のサイン
#コンテンツ  — カードの内容・質に関する観察
#UX          — 操作・画面に関する観察
```

# 収集ソースと方法

```
ソース1: ユーザーインタビュー（USER_TEST_SCRIPT.md 参照）
  タイミング: MVP 公開後の5人テスト / 1週間後フォロー
  方法: 対面 or ビデオ通話
  記録: テスト当日中にここへ追記

ソース2: 1週間後フォローアップ
  タイミング: 初回テストから7日後
  方法: LINE / メール / 電話
  注目: 「この1週間で畑を見ましたか？」への回答

ソース3: SNS の自発的投稿
  タイミング: 随時
  方法: #ImagingAgriculture を検索
  記録: 投稿を引用してインサイトを添える

ソース4: アプリ内の行動データ（PostHog）
  タイミング: 週次
  方法: FOUNDER_DASHBOARD.md の週次確認で発見したパターン
  記録: 定量データから読み取れる「真実」を言語化して記録
```

---

# 確認・活用ルール

```
□ ユーザーと接触するたびに当日中に追記する
□ 週次で読み返して新しいインサイトがないか確認する
□ 機能開発の前にここを読んで「ユーザーの真実」を思い出す
□ 失敗シグナルが3件以上たまったら体験設計を見直す
□ 「行動変化」が出始めたら Phase 進行の判断材料にする
```

---

# Phase 進行判断への活用

```
Phase 0 → Phase 1 への移行条件:
  □ ★★★シグナル（行動変化）が 3件以上
  □ 失敗シグナルが ★★★ より少ない
  □ D30 Retention 10% 以上（定量）

Phase 1 → Phase 2 への移行条件:
  □ 「雑草の見方が変わった」発言が複数件
  □ 「もっとカードが見たい」発言が複数件（= Weed Cards 需要）
  □ MAU 100人以上（定量）
```
