Imaging Agriculture

Version 1.0

# Purpose

ユーザーが実際に言ったこと・見せた行動を記録する。

推測ではなく観察を蓄積する。

このリポジトリは

* 仕様変更の根拠
* コンテンツ優先度の根拠
* North Star の妥当性検証

に使う。

# Entry Format

各エントリは以下の形式で記録する。

```
## [Entry ID] [日付] [ソース]

### Observation

何が起きたか・何を言ったか（事実のみ）

### Quote

発言をそのまま記録する
（要約しない）

### Insight

この観察から何が言えるか

### Decision

このインサイトを受けてドキュメントや仕様をどう変えるか
（変えない場合はその理由も記録する）

---
```

# Entry Examples

## CTR-001 2024-01-01 インタビュー

### Observation

農業参入希望者（30代・会社員）に
「土地を見たときに何を考えますか？」と質問した。

### Quote

「正直、何を見ればいいかわからないです。
草が生えてるな、くらいしか思わない。
農家の人はあの草を見て何か読み取ってるんですよね？
それを知りたいんですけど、どこで教えてくれるのかも知らなくて。」

### Insight

「土地を見る目がない」という課題は本人も自覚している。
かつ「農家は読んでいる」という認識がある。
ギャップを埋めたいという動機が存在する。

### Decision

PRODUCT_THESIS の Problem セクションの根拠として記録。
コンテンツの切り口は「農家が見ていること」を体験させる方向で正しい。

# Signal vs Noise

記録するもの

* 繰り返し出てくる言葉・表現
* 行動（言葉でなく実際にやったこと）
* 感情的な反応（驚き・喜び・フラストレーション）

記録しないもの

* 「〜があればいいのに」という要望（Feature Request）
  → 要望は ROADMAP に記録する
* 解釈・推測
  → Insight セクションに分離する

# Privacy

実名は記録しない。

発言の転記には本人の同意を得る。

SNS 発言の収集は公開投稿のみ対象とする。
