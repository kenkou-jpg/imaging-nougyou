# MVP Success Criteria

Imaging Agriculture

Version 1.0

---

# Purpose

「何をもってMVP成功とするか」を定義する。

ダウンロード数・PVではない。

ユーザーの土地を見る視点が変わったか。
土地を見る行動が生まれたか。

これを成功とする。

---

# North Star

Land Reading Sessions

カードを開き → 回答し → 結果画面まで到達した回数

---

# MVP Phase 0 成功条件

## ミニマム成功

```
□ 30人以上が利用した
□ 回答完了率 70% 以上
```

この2つが達成されれば Phase 1 へ進む。

---

# Success Metrics

## 1. Engagement（関与）

| 指標 | 定義 | MVP目標値 | 達成判定 |
|------|------|----------|---------|
| DAU | 当日に1回以上 Land Reading Session を完了したユーザー数 | 5人以上（Phase 0） | □ |
| 平均セッション時間 | 1セッションあたりの滞在時間（land_card_viewed 〜 result_viewed） | 2分以上 | □ |
| 平均閲覧カード数 | 1セッションあたりの完了カード数 | 2枚以上 | □ |
| WAU | 週単位アクティブユーザー数 | 10人以上（Phase 0） | □ |

### 判断基準の補足

平均セッション時間 2分以上 = 写真を「眺めている」証拠
2分未満 = 流し見・クイズ感覚で使われている可能性

---

## 2. Completion（完了率）

| 指標 | 定義 | MVP目標値 | 達成判定 |
|------|------|----------|---------|
| カード回答完了率 | land_card_viewed に対する land_card_answered の割合 | 80% 以上 | □ |
| Result画面到達率 | land_card_answered に対する result_viewed の割合 | 90% 以上 | □ |
| 解説読了率 | result_viewed 後に knowledge_card_opened または next_card に進んだ割合 | 50% 以上 | □ |
| 途中離脱率 | land_card_viewed 後に回答せず離脱した割合 | 20% 以下 | □ |

### 失敗シグナル

途中離脱率 40% 超 → 写真または問いに問題がある可能性
Result 画面で即離脱 → 解説が読まれていない可能性

---

## 3. Retention（継続率）

| 指標 | 定義 | MVP目標値 | 達成判定 |
|------|------|----------|---------|
| D1 Retention | 初回完了翌日に再訪した割合 | 30% 以上 | □ |
| D7 Retention | 初回完了から7日後に再訪した割合 | 20% 以上 | □ |
| D30 Retention | 初回完了から30日後に再訪した割合 | 10% 以上 | □ |
| Streak 継続率 | 3日以上連続利用しているユーザーの割合（アクティブユーザー中） | 20% 以上 | □ |

### 判断基準の補足

D1 < 20% → 初回体験に問題がある可能性。オンボーディングを見直す
D7 < 10% → 継続の動機が生まれていない。コンテンツを増やす or 体験を深める
D30 > 10% → 習慣化が始まっている。Phase 1 へ進む十分な根拠

---

## 4. User Insight（ユーザー洞察）

定性的な成功指標。数値では測れないが最も重要。

### 収集方法

USER_TEST_SCRIPT.md に基づくインタビュー
Discord / SNS での発言収集
CUSTOMER_TRUTH_REPOSITORY.md への記録

---

### 成功シグナル（発言の例）

以下の発言が1件でも出たら強い成功シグナル。

```
「旅行中に畑を見るようになった」
「雑草を見るのが楽しくなった」
「空き地を見る目が変わった」
「ここなら何を育てたいか考えるようになった」
「土地を歩いてみたくなった」
「外に出たくなった」
「また土地を見たくなってきた」
「農業に詳しくなくても楽しめた」
```

---

### 中立シグナル（要観察）

```
「面白かった」→ 何が面白かったかを掘り下げる
「また使いたい」→ なぜ使いたいかを確認する
「写真が綺麗だった」→ 観察力が育ったかも確認する
```

---

## 5. Behavioral Change（行動変化）

最重要カテゴリ。

### 測定方法

1週間後のフォローアップインタビュー
（USER_TEST_SCRIPT.md の「1週間後」セクション参照）

---

### 行動変化の成功指標

| 行動変化 | 成功シグナル | 測定方法 |
|---------|------------|---------|
| 畑を見る回数が増えた | 「気づいたら農地を見ていた」 | インタビュー |
| 雑草を見る回数が増えた | 「散歩中に草の種類が気になった」 | インタビュー |
| 土地を想像するようになった | 「空き地を見て植えるものを考えた」 | インタビュー |
| 土地に関連した行動を取った | 「農地を見に行った」「土をさわった」 | インタビュー |

---

### 最高の成功状態

アプリを閉じた後も、
土地を見る習慣が日常に組み込まれること。

これが North Star「Land Reading Sessions」の
真のゴール。

# Failure Criteria（失敗判定）

以下のいずれかが発生したら失敗と判断し、
根本的な見直しを行う。

## 体験の失敗

```
□ 「農業辞典になっている」（知識の羅列）
□ 「クイズアプリになっている」（正解・不正解の緊張感）
□ 「学習アプリになっている」（義務感・負担感）
□ 「土地を見る行動変化が起きていない」（D30後も視点変化なし）
```

## 数値の失敗

```
□ 回答完了率 50% 未満
□ D1 Retention 10% 未満
□ 平均セッション時間 1分未満
□ 5人中4人が「勉強になった」と発言
```

## コンテンツの失敗

```
□ 「問いが難しすぎて答えられない」が3人以上
□ 「写真が何なのかわからない」が複数発生
□ 「解説を読まない」が70%以上
```

# 成功後の次のステップ

MVP Phase 0 成功 →

```
1. ROADMAP Phase 1 へ進む
2. 雑草カードを追加（コンテンツ拡充）
3. D30 Retention を改善する施策を実施
4. 5人 → 30人 → 100人へのスケール計画
```

MVP Phase 0 失敗 →

```
1. 失敗した項目の原因を CUSTOMER_TRUTH_REPOSITORY.md に記録
2. 最も重要な1点だけ修正する
3. 再度5人でテスト
4. 「削る」選択肢を常に持つ
```
